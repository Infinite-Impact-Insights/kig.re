Let's break this down: W — is the number of processes.

Since MRI Ruby can only max out a single CPU core, one process limits us to a single core. Therefore we typically want to run the same number of processes as we have CPU cores available to us.

If we are not able to push each process to the max, we might need to use more processes than we have cores available, but that is typically a sign that number of threads is insufficient.

This brings to the last value: T.

To properly determine how many threads per process we should run, we'd need some sort of monitoring like DataDog or NewRelic (it can also be done without the monitoring giants, but it's much more involved).

What we want to know is how much time each process (running on 1 thread) spends in CPU versus IO. That gives us the ratio to figure out the T in question.
